# The Story Nexus 1.0.2

GitHub release title: `The Story Nexus 1.0.2`

Recommended tag: `1.0.2`

Reviewed through commit: `721d783`

Compare range after tagging: `1.0.1...1.0.2`

## Release Summary

The Story Nexus 1.0.2 is an editor-focused maintenance and workflow release. It replaces the old Lexical Playground integration with a smaller purpose-built writing editor, improves SceneBeat behavior and performance, and adds Playwright regression coverage for the editor and local LLM workflows.

This release is especially important for writers using inline SceneBeat generation, autosaved chapter content, and local LM Studio-backed generation.

## Highlights

- Replaced the large embedded Lexical Playground with a focused `mainLexicalEditor` built for The Story Nexus writing workflow.
- Added a cleaner editor toolbar, floating AI rewrite controls, chapter content persistence, word count, lorebook highlighting, slash command support, asset image insertion, and plain-text serialization helpers.
- Improved SceneBeat blocks with inline command editing, generation lifecycle handling, prompt and diagnostics dialogs, matched context display, regeneration support, and persisted SceneBeat snapshots.
- Fixed slow chapter loading when SceneBeat text was present.
- Removed the unused Lexical Playground source tree and assets to reduce maintenance burden and bundle surface area.
- Added Playwright E2E coverage for editor loading, serialized Lexical state, SceneBeat insertion/removal, and autosave persistence.
- Added local LLM E2E coverage through LM Studio, including health checks, local-only generation, paid-provider network guards, and multi-SceneBeat acceptance flow.
- Added repository working rules and Playwright testing documentation for future editor and local AI changes.

## User-Facing Changes

### Editor

- The app remains editor-first, but the writing surface now uses the app-specific Lexical editor instead of the general-purpose playground bundle.
- Seeded story content loads as structured paragraphs instead of one large block.
- Editor autosave behavior is covered by E2E tests and should preserve chapter content more reliably.
- Inline SceneBeat blocks now have clearer lifecycle handling for command input, generation, acceptance, regeneration, and removal.

### SceneBeats

- SceneBeat text no longer causes the same long loading delays seen in the previous workflow.
- SceneBeat generation state is tracked with serialized snapshots so chapter persistence can keep inline AI blocks consistent.
- SceneBeat UI pieces are split into smaller components, making future fixes and improvements easier.

### Local AI Testing

- The local LLM test lane is explicitly designed for LM Studio at `http://localhost:1234`.
- Local LLM tests abort when LM Studio is unavailable or no model is loaded.
- E2E tests block paid-provider endpoints during local LLM coverage, including OpenAI, OpenRouter, NanoGPT, and Google AI hosts.
